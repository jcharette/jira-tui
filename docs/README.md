# Project Docs

This folder contains the durable project documentation for Jira TUI.

## Start Here

- [project-state.md](project-state.md): current architecture, commands, behavior, and assumptions.
- [security.md](security.md): authentication, local storage, diagnostics, and integration security model.
- [install.md](install.md): release, Go, source, and Homebrew install paths.
- [roadmap.md](roadmap.md): current product direction and known future work.
- [backlog.md](backlog.md): curated backlog index linked to GitHub Issues.
- [working-agreement.md](working-agreement.md): documentation and release maintenance rules.
- [releases/CHANGELOG.md](releases/CHANGELOG.md): high-level release history.
- [decisions/](decisions): architecture and product decisions we do not want to re-litigate.

## Update Rules

Docs are part of the definition of done. Update docs in the same change when behavior, architecture,
security posture, release process, or user-visible workflows change.

Use GitHub Issues as the public backlog source of truth. Keep [backlog.md](backlog.md) short and
focused on sequencing/context rather than duplicating full issue bodies.
